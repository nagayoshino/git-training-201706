Gitについて調べたこと。

コマンドまとめ
・git status：現在のステータスを表示
・git add [ファイルパス]：ファイルを追加
　* git add -n：追加されるファイルを調べる
  * git add -u[ファイルパス]：変更されたファイルを追加する(新規ファイルは対象外)
  * git rm --cached [ファイルパス]：addしたファイルを除外
・git commit ：ファイルをコミット
・git push ：自分の更新内容をリモートに公開
・gitk --all ：現在のツリーとブランチを表示
・git branch ：自身のブランチ情報の表示、作成
・git show-branch :ブランチの履歴の表示
・git clone：既存リポジトリのコピーを作る
・git init：リポジトリを作成する
・git fetch：フォルダ内のファイルが更新されるわけではなく、gitkのファイルが更新される
・git clone：既存リポジトリのコピーを作る
・git merge[マージ対象のブランチ名]：ブランチファイルとコミットファイルをマージする
・git pull：他のリポジトリの変更点をローカルとマージする
・git checkout[ファイルパス]：更新した部分を取り消し、masterまで戻る
・git log：コミットログを閲覧する

コミットの流れ
1. add
2. commit
3. push

マージの流れ
1. add
2. commit
3. fetch
4. F5(gitk --all)
5. merge/pull
6. ファイルを再度開き、マージ後の内容に整える
7. add
8. commit/ merge --continue / pull -- continue
7. push

その他
コマンドの末尾に --helpを記入することでコマンドのオプションを一覧で表示できる。

※コマンド以外の内容もぜひ書いて下さい。※
-------------------------------------------------------
◎コマンド詳細
・git status
　前回のコミットと比較してどのファイルが変更されたかを表示します
　-s オプションを付けると、説明文を表示しないようにできます
　さらに - bオプションを付けることで、説明文は表示せずブランチ名だけを表示できるようにします
　



======================================================

Git：ファイル履歴を管理するソフトウェア。OSSとして開発されている
　　 最近はMicrosoftも開発に参加している
　　 もっとも使われている分散バージョン管理システム（DVCS) 。

VCS：バージョン管理システム(VersionControlSystem)
　　 主にファイルのVCSを指し、仮想マシン(VM)のスナップ管理などもVMに閉じたVCSを提供していると捉える。


　　・Local：RCS
　　・Center：SVN
　　・Distributed：Git

リポジトリ：ファイルの置き場所で、VCS全体で使える言葉。コミットやリビジョンとほとんど同じだが、ブランチとは違う
　　　　　　initで作成、cloneで複製することで、ローカルリポジトリを作成できる
